# 🚀 双击即用 - 打包说明

## ✨ 优势

- ✅ **无需安装 Node.js** - 用户电脑不需要任何依赖
- ✅ **双击即用** - 直接运行，无需配置
- ✅ **独立运行** - 包含所有必要文件
- ✅ **跨平台** - 支持 macOS、Windows、Linux

## 📦 打包步骤

### 第一步：安装打包工具（仅开发者需要）

```bash
npm install
```

### 第二步：打包应用

#### macOS
```bash
npm run build:mac
```
生成文件：
- `dist/秒著.dmg` - 安装包（推荐分发）
- `dist/秒著-mac.zip` - 压缩包

#### Windows
```bash
npm run build:win
```
生成文件：
- `dist/秒著 Setup.exe` - 安装程序（推荐分发）
- `dist/秒著.exe` - 便携版（无需安装，直接运行）

#### Linux
```bash
npm run build:linux
```
生成文件：
- `dist/秒著.AppImage` - 便携版（无需安装，直接运行）
- `dist/秒著.deb` - Debian/Ubuntu 安装包

## 🎁 分发方式

### 方式一：便携版（推荐）
- **Windows**: 分发 `秒著.exe`，用户双击直接运行
- **Linux**: 分发 `秒著.AppImage`，用户双击直接运行
- **macOS**: 分发 `秒著.app`（从 dmg 提取），用户双击运行

### 方式二：安装包
- **Windows**: 分发 `秒著 Setup.exe`，用户安装后使用
- **macOS**: 分发 `秒著.dmg`，用户拖拽安装
- **Linux**: 分发 `秒著.deb`，用户安装

## 📋 用户使用步骤

### Windows 用户
1. 下载 `秒著.exe` 或 `秒著 Setup.exe`
2. 双击运行（便携版）或安装（安装版）
3. 开始使用

### macOS 用户
1. 下载 `秒著.dmg`
2. 双击打开，拖拽到应用程序文件夹
3. 开始使用

### Linux 用户
1. 下载 `秒著.AppImage`
2. 右键 → 属性 → 权限 → 勾选"允许作为程序执行"
3. 双击运行

## 🔧 高级配置

### 自定义应用图标
1. 准备 512x512 的 PNG 图标
2. 转换为各平台格式：
   - macOS: `icon.icns`
   - Windows: `icon.ico`
   - Linux: `icon.png`
3. 放在项目根目录
4. 重新打包

### 修改应用信息
编辑 `package.json` 中的 `build` 部分：
```json
{
  "build": {
    "appId": "com.yourcompany.miaozhu",
    "productName": "秒著",
    "copyright": "Copyright © 2024 Your Company"
  }
}
```

## 💡 提示

1. **首次打包**：需要下载 Electron，可能需要几分钟
2. **文件大小**：约 150-200MB（包含所有依赖）
3. **杀毒软件**：可能误报，需要添加信任
4. **macOS 签名**：未签名的应用需要在"系统偏好设置 → 安全性与隐私"中允许

## 🎯 推荐分发方案

### 给普通用户
- Windows: `秒著 Setup.exe`（安装版）
- macOS: `秒著.dmg`（标准安装）
- Linux: `秒著.deb`（系统安装）

### 给高级用户
- Windows: `秒著.exe`（便携版）
- macOS: `秒著.app`（便携版）
- Linux: `秒著.AppImage`（便携版）

## 📊 打包后的文件结构

```
dist/
├── 秒著 Setup.exe        # Windows 安装程序
├── 秒著.exe              # Windows 便携版
├── 秒著.dmg              # macOS 安装镜像
├── 秒著-mac.zip          # macOS 压缩包
├── 秒著.AppImage         # Linux 便携版
└── 秒著.deb              # Linux 安装包
```

## ⚠️ 注意事项

1. **Windows SmartScreen**：未签名的应用可能被拦截
   - 解决：点击"更多信息" → "仍要运行"

2. **macOS Gatekeeper**：未签名的应用无法直接打开
   - 解决：右键点击 → "打开" → "打开"
   - 或在"系统偏好设置 → 安全性与隐私"中允许

3. **Linux AppImage**：需要添加执行权限
   - 解决：右键 → 属性 → 权限 → 允许执行

打包完成后，你可以：

2. 用户无需安装 Node.js
3. 用户双击即可使用
